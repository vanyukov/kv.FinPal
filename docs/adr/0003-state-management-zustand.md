# 3. State Management: Zustand vs Redux Toolkit

Date: 2026-01-06

## Status

Accepted

## Context

Необходимо выбрать решение для управления состоянием приложения. Рассматривались Redux Toolkit и Zustand.

## Decision

Выбран **Zustand** для управления состоянием приложения.

## Rationale

### Преимущества Zustand
1. **Простота** - минимальный boilerplate, интуитивный API
2. **Производительность** - легковесная библиотека (~1KB)
3. **Гибкость** - можно использовать без провайдеров
4. **TypeScript** - отличная поддержка типов из коробки
5. **Персистентность** - встроенная поддержка через middleware

### Недостатки Redux Toolkit
1. **Boilerplate** - больше кода для простых случаев
2. **Сложность** - избыточна для большинства задач проекта
3. **Размер** - больше размер bundle

## Consequences

### Положительные
- Меньше кода для поддержки
- Быстрее разработка новых фич
- Проще обучение новых разработчиков
- Лучшая производительность приложения

### Негативные
- Меньше экосистемы по сравнению с Redux
- Меньше готовых решений и middleware

## Alternatives Considered

- **Redux Toolkit** - более зрелая экосистема, но избыточна для проекта
- **Jotai** - атомарное управление состоянием, но сложнее для понимания
- **Context API** - недостаточно для сложного состояния приложения
