# 2. Выбор технологического стека

Date: 2026-01-06

## Status

Accepted

## Context

Необходимо выбрать технологический стек для разработки PWA приложения FinPal с оффлайн-first подходом.

## Decision

Выбран следующий стек:

### Frontend
- **React 19** - современная версия React с улучшенной производительностью
- **TypeScript** - типобезопасность и лучшая поддержка IDE
- **Vite** - быстрая сборка и HMR
- **Material-UI** - готовая компонентная библиотека с доступностью
- **React Router** - навигация и защищенные маршруты
- **Zustand** - легковесное state management
- **Recharts** - визуализация данных

### Backend/Infrastructure
- **Firebase (Firestore, Auth, Functions, Hosting)** - бессерверная архитектура
- **Firebase persistence** - оффлайн-персистентность
- **IndexedDB** - локальное хранилище для очереди синхронизации

### Валидация
- **Zod** - end-to-end валидация и генерация типов

### CI/CD
- **GitHub Actions** - автоматизация сборки и деплоя

### Мониторинг
- **Sentry** - отслеживание ошибок
- **Cloud Logging** - структурированное логирование

## Consequences

### Положительные
- Быстрая разработка благодаря готовым решениям
- Типобезопасность на всех уровнях
- Оффлайн-first архитектура из коробки
- Масштабируемость через Firebase

### Негативные
- Зависимость от Firebase экосистемы
- Потенциальные ограничения Firestore (квоты)
- Необходимость изучения новых технологий командой

## Alternatives Considered

- **Redux Toolkit** вместо Zustand - выбран Zustand из-за простоты и меньшего boilerplate
- **Next.js** вместо Vite + React - выбран Vite для более простой настройки PWA
- **MongoDB + Express** вместо Firebase - выбран Firebase для бессерверной архитектуры
