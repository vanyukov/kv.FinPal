# 1. Record architecture decisions

Date: 2026-01-06

## Status

Accepted

## Context

Нам нужно записывать архитектурные решения, принятые в этом проекте. Любое лицо, работающее над проектом, должно иметь возможность понять, почему было принято определенное решение.

## Decision

Мы будем использовать Architecture Decision Records (ADR), как описано Майклом Нейгардом в этой статье: http://thinkrelevance.com/blog/2011/11/15/documenting-architecture-decisions

Мы будем хранить ADR в виде Markdown файлов в директории `docs/adr/`.

## Consequences

Архитектурные решения будут документированы и доступны для всех участников проекта. Это поможет новым разработчикам понять контекст решений и избежать повторного обсуждения уже решенных вопросов.
